被动流量分析

流量镜像分析

通过交换机/路由器端口镜像或TAP设备，将网络流量复制到分析设备，不影响网络性能。

优势

  • 不影响网络性能
  • 可实时分析
  • 适合大规模部署

局限

  • 单向流量（无响应）
  • 可能丢包
  • 加密流量难以解析

NetFlow/sFlow采集

NetFlow是思科提出的流量统计协议，可以采集流量元数据进行分析。

NetFlow关键字段
# 采集的元数据信息
- 源/目的IP地址
- 源/目的端口
- 协议类型
- 流量字节数
- 数据包数量
- 会话时长
- TOS/DSCP值
- 下一跳信息

# 优势
- 低资源消耗
- 可采集大规模流量统计
- 支持历史分析

DPI深度包检测

DPI技术可以解析到数据包的应用层内容，用于识别协议和应用。

可识别内容

  • 应用协议（HTTP、DNS、SMTP等）
  • 攻击特征（SQL注入、XSS等）
  • 恶意软件通信特征
  • 敏感数据外传

局限性

  • 加密流量无法解析
  • 资源消耗较高
  • 隐私合规问题

主动流量分析

流量劫持与重定向

通过ARP欺骗或ICMP重定向，将流量引入分析设备。

ARP欺骗检测原理
# 正常ARP表
192.168.1.1 -> aa:bb:cc:dd:ee:ff (网关)
192.168.1.100 -> 11:22:33:44:55:66 (正常主机)

# 异常ARP表（可能存在ARP欺骗）
192.168.1.1 -> aa:bb:cc:dd:ee:ff
192.168.1.1 -> 11:22:33:44:55:66  # 双重映射！

# 检测方法
- 检查ARP缓存中的重复MAC
- 监控网关MAC是否频繁变化
- 使用静态ARP绑定

代理与蜜罐分析

部署代理服务器或蜜罐诱捕攻击者，分析攻击行为。

代理分析

  • 记录完整请求/响应
  • 可修改请求内容
  • 适合Web攻击分析

蜜罐分析

  • 诱捕攻击者
  • 分析攻击工具
  • 提取恶意样本

行为分析

基线学习与异常检测

通过学习正常流量基线，检测偏离正常模式的异常行为。

基线分析维度
# 时间维度
- 每日/每周流量趋势
- 业务高峰时段
- 异常时间段流量

# 空间维度
- 常见通信IP/端口
- 地理流量分布
- 内部互访关系

# 内容维度
- 常用协议分布
- 平均包大小
- 会话时长分布

# 检测方法
- 统计阈值（均值+标准差）
- 机器学习异常检测
- 聚类分析

会话与连接分析

分析TCP/UDP会话的建立、维持和结束过程。

  • 新建连接数：突增可能意味着扫描或攻击
  • 会话时长：过长的会话可能为后门连接
  • 连接失败率：高失败率可能是暴力破解
  • 数据传输量：异常大的数据外传需要关注

加密流量分析

即使无法解密加密流量，仍可通过元数据发现异常。

JA3/JA3S指纹

TLS握手指纹，可识别客户端/服务器

SNI特征

HTTPS请求中的域名信息

流量模式

包大小、时间间隔、传输速率

关联分析

多源数据关联

将网络流量日志与系统日志、应用日志等进行关联分析。

关联分析示例
# 攻击链路还原
1. Web访问日志 -> 发现SQL注入尝试
2. 网络流量 -> 确认注入payload发送
3. 数据库日志 -> 确认注入执行结果
4. 系统日志 -> 确认攻击者获得权限
5. 认证日志 -> 发现异常登录

# 关联字段
- 时间戳（精确到毫秒）
- IP地址
- 用户名
- Session ID
- 请求ID/Trace ID

IOCs指标提取

从分析过程中提取可执行的威胁指标。

恶意指标

  • IP地址（C2服务器）
  • 恶意域名
  • 文件哈希（MD5/SHA1/SHA256）
  • URL模式

可疑指标

  • 异常用户行为
  • 非授权时间登录
  • 敏感文件访问
  • 异常网络连接

情报驱动分析流程

分析流程
# 1. 情报收集
从内部告警、威胁情报平台、行业共享收集情报

# 2. 情报富化
将IOCs与流量日志进行匹配
补充情报背景信息（攻击者、动机、手法）

# 3. 主动狩猎
基于已知TTPs主动搜索潜在威胁
验证现有安全设备告警

# 4. 响应处置
确认威胁后快速隔离受影响主机
更新防护规则

# 5. 复盘改进
分析检测遗漏原因
优化检测规则
